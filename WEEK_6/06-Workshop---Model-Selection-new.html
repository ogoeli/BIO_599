Introduction

In this workshop, we will explore model selection and validation using the mammals.csv dataset.

Research question: What factors predict how long mammals sleep?
We will use data on 39 species of mammals from 7 different orders, with Order included as a random intercept.

We will work through four approaches:

  1. Likelihood Ratio Tests for Minimum Adequate Model using a structured 10-step protocol.
  2. Stepwise AIC (demonstrated but not recommended).
  3. Model averaging to incorporate model uncertainty.
  4. Cross-validation to assess predictive performance.

Explore relationships between predictors

Before modeling, its important to look for collinearity between predictors.
Here we use GGally::ggpairs to visualize scatterplots, correlations, and distributions for each variable.

GGally::ggpairs(mammalsc)

Predictor variables will almost always be correlated in observational studies.
When these correlations are strong, it can be difficult to estimate parameters in a multiple regression model.
This can lead to coefficients with large standard errors and unstable model estimates.

When deciding which predictors to include, consider:

  • Are the predictors ecologically relevant?
  • Are they feasible to collect in other studies?
  • Are they closer to the mechanism underlying the response?
  • Rule of thumb: have at least 1020 observations per predictor in regression.

Strategies for handling collinearity

  • Keep all variables if the focus is on prediction and collinearity does not cause instability.
  • Include collinear variables if estimating causal effects and they each have separate direct effects on the response.
  • Drop one variable from a set of highly correlated predictors (e.g., correlation > 0.7).
  • Compare univariate regressions and choose the predictor with the strongest relationship to the response.
  • Stepwise model selection (often drops collinear variables automatically but be cautious, this can lead to overfitting).
  • Principal Components Analysis (PCA) to combine correlated variables into composite predictors.
  • Structural Equation Modeling (SEM) to model complex relationships explicitly.

1. Likelihood Ratio Tests for Minimum Adequate Model (10-Step Protocol)

We will follow this process:

  1. Fit a linear regression.
  2. Fit a GLS model (baseline for comparison).
  3. Choose variance structure (if needed).
  4. Fit the mixed-effects model (REML).
  5. Compare models (GLS vs LME).
  6. Validate model (diagnostics).
  7. Find optimal fixed effects drop interactions first, then main effects.
  8. Repeat selection until the best fixed structure is found.
  9. Refit with REML and validate.
  10. Interpret results.

2. Stepwise AIC

Using stepwise elimination or addition of terms is a common practice but also highly criticized.

Stepwise model selection involves:

  • Fitting a multiple regression with many variables.
  • Iteratively adding or deleting terms, refitting the model each time.
  • Continuing until only statistically significant terms remain.

How it works

  • MASS package: stepAIC() function (direction = "backward", "forward", or "both").
  • lmerTest package: step() function (can step through both random and fixed effects).

Note: A Likelihood Ratio Test (LRT) of full vs reduced models is not the same as backward stepwise selection.
drop1() drops one term at a time and tests each against the full model.
The step() or stepAIC() function will drop the worst term (or add the best), then repeat the process iteratively.

Backward selection:

  1. Fit a full model containing all predictors of interest.
  2. Consider all possible models formed by dropping one of these predictors.
  3. Keep the current model, or drop the worst predictor depending on:
    • Highest p-value (> 0.05)
    • Adjusted R (higher is better) for lm
    • AIC (lower is better) for lme
  4. Repeat until no improvement is possible.

Forward selection:

  1. Fit a null model containing no predictors.
  2. Consider all possible models formed by adding 1 predictor at a time.
  3. Keep the current model, or add the predictor based on comparisons between nested models.
  4. Repeat until no improvement is possible.

Here, we will use the same model as before with life_span, gestation, log(body_wt), and danger, and run stepAIC().

The default direction is backward selection.

Reading the output

  • The first table shows the current best model (<none>), indicating no variables have been dropped yet, compared to all possible models with one fewer predictor.
  • Sorted from lowest (best fitting) to highest (worst fitting) AIC.
  • If removing a variable results in a lower AIC, it is dropped.

For example:
- Removing life_span might lower the AIC from 261 to 260 stepAIC will drop it.
- The final model keeps the remaining predictors because removing any of them would increase AIC.

But is a small change in AIC meaningful?

An AIC change from 261.67 to 260.02 is technically better but is the difference meaningful?
Rules of thumb for meaningful AIC change: AIC of 2, 5, or even 10.

Key takeaways:

  • stepAIC() is more conservative than some backward selection methods less likely to drop variables quickly.
  • It may keep more predictors than backward selection by LRT, leading to more complex models.
  • The results are not the same as the LRT-based full vs reduced model comparisons.
  • For inference, LRT-based model selection is generally preferred.
  • For exploration, stepwise selection can be used but not if you claim it is hypothesis testing!
  • Always be transparent about your method and why you chose it.

3. Model Averaging (Multi-Model Inference)

The Akaike Information Criterion (AIC) is often used to choose among competing models, not just nested models.
AIC balances model fit with model complexity the best model has the lowest AIC.

Why use model averaging?

Rather than selecting a single best model, we can average predictions or parameter estimates among multiple good models.
This incorporates model uncertainty into our estimates by weighting each models parameters by that models Akaike weight.

Steps for model averaging

  1. Write down all biologically plausible models (or fit all subsets of predictors).
  2. Fit each model and calculate AIC for each.
  3. Compute Akaike weights from AIC values these reflect the relative plausibility of each model.
  4. Identify the model set based on a AIC threshold (commonly 4 or 6 from the best model).
  5. Calculate weighted predictions and SEs that reflect both model uncertainty and sampling uncertainty.

How it works in practice

Here, we use the MuMIn::dredge() function, which returns all possible subsets of the starting model, ranked by AIC.

Interpreting averaged coefficients

The output shows two sets of coefficients:

  • Full average: Assigns 0 to predictors not present in a given model before averaging shrinks weak parameters toward zero (more conservative; recommended for interpretation).
  • Conditional average: Averages only across models where the predictor is included can give larger effect sizes for predictors with low support.

Best practice: Use the full average for inference, as it guards against overestimating the importance of weak predictors.

Key points

  • Exploratory use: All-subsets selection of huge models (with many predictors/interactions) can overfit and include uninformative parameters.
  • Inference use: Limit candidate models to a small set that reflect clear, biologically motivated hypotheses.
  • Be explicit in your methods about the AIC threshold and model selection process.
  • In our example, model averaging suggests that only danger is consistently important across plausible models.

4. Cross-Validation

To evaluate the performance of a model, we need to measure how well the predictions made by the model match the observed data.

There are several common cross-validation designs:

  1. Refit the model on a new dataset
    • Train model on one dataset
    • Test model predictions on a completely independent dataset
  2. Split data into training and testing datasets
    • Randomly subset a percentage (e.g., 50%) of observations
    • For grouped data (with random effects):
      • Train model on all data from a subset of groups (e.g., 6 sites), test on the remaining group
      • Alternatively, split data within each group so part goes to training and part to testing
  3. Leave-one-out cross-validation (LOO)
    • Train model on all but one observation, test on the held-out observation
    • Repeat for every observation in the dataset

k-fold cross-validation

A common approach is k-fold CV:

  • Partition the data into k subsets (folds)
  • Train the model on k-1 folds, test on the remaining fold
  • Repeat until each fold has served as the test set once
  • Pool results to estimate model fit (e.g., cross-validated R or RMSE)

Repeated k-fold CV reduces variability in the evaluation and is useful for small datasets.

When to use which approach

  • Small datasets Repeated k-fold CV is more reliable
  • Stable model performance & large dataset Regular k-fold CV may be sufficient
  • LOO CV Every observation is tested, but can be computationally expensive

Interpreting CV metrics

  • R: Squared correlation between observed and predicted values. Higher = better.
    • Marginal R (from performance()): proportion of variance explained by fixed effects only.
    • Conditional R: proportion explained by both fixed and random effects.
    • Crossvalidated R (from performance_cv()): how much variance is explained when predicting unseen folds.
  • RMSE: Root Mean Squared Error average prediction error, in the same units as the response. Lower = better.
  • MSE: Mean Squared Error same as RMSE but squared; more sensitive to large errors.

What is good?
- Depends on the scale of your outcome and how much error is acceptable.
- Compare CV metrics to insample metrics:
- If CV R is much lower than insample R model may not generalize well (possible overfitting).
- If CV RMSE is much higher than insample RMSE predictions are less accurate on new data.

Example from our model:
- Insample: Marginal R 0.40, RMSE 3.36 hours model explains ~40% of variance, with moderate prediction error on training data.
- Crossvalidated: R 0.13, RMSE 3.8 hours when predicting unseen folds, variance explained drops sharply, and prediction error increases slightly.
- This gap suggests the models fixed effects have limited predictive power for new observations it may be overfitting or missing important predictors.

MSE/RMSE rules of thumb:
- MSE close to 0 good fit.
- MSE increases exponentially with error size.
- RMSE is easier to interpret because its in the same units as the response variable.

Grouped k-fold cross-validation for mixed models

When working with mixed models, we must avoid splitting data from the same group into both training and test sets.
Here, well implement manual grouped kfold cross-validation so that all observations from a given order are assigned to the same fold.

library(lme4)

set.seed(123)

# Number of folds
k <- 5

# Get unique groups and randomly assign them to folds
groups <- unique(mammalsc$order)
fold_assignments <- sample(rep(1:k, length.out = length(groups)))
names(fold_assignments) <- groups

# Data frame to store results
cv_results <- data.frame(Fold = integer(), R2 = numeric(), RMSE = numeric())

# Loop through folds
for (fold in 1:k) {
  
  # Identify test and train groups
  test_groups <- names(fold_assignments[fold_assignments == fold])
  train_data <- mammalsc[!mammalsc$order %in% test_groups, ]
  test_data  <- mammalsc[mammalsc$order %in% test_groups, ]
  
  # Fit the mixed model on the training data
  model <- lmer(total_sleep ~ life_span + gestation + log(body_wt) + danger + (1|order),
                data = train_data)
  
  # Predict on the test data
  preds <- predict(model, newdata = test_data, allow.new.levels = TRUE)
  
  # Calculate R and RMSE for this fold
  ss_res <- sum((test_data$total_sleep - preds)^2)
  ss_tot <- sum((test_data$total_sleep - mean(test_data$total_sleep))^2)
  r2 <- 1 - (ss_res / ss_tot)
  rmse <- sqrt(mean((test_data$total_sleep - preds)^2))
  
  # Store results
  cv_results <- rbind(cv_results, data.frame(Fold = fold, R2 = r2, RMSE = rmse))
}

# Average performance across folds
cv_summary <- data.frame(
  Mean_R2 = mean(cv_results$R2),
  SD_R2   = sd(cv_results$R2),
  Mean_RMSE = mean(cv_results$RMSE),
  SD_RMSE   = sd(cv_results$RMSE)
)

This approach:
- Respects the grouping structure in your random effect (order)
- Gives you a realistic assessment of how well the model predicts on entirely unseen groups
- Allows you to report both foldlevel metrics and the average performance across folds

Interpreting grouped CV results

The cv_results table shows foldlevel performance for each test set (all species from one or more order values):

  • R indicates how much variance the model explains in that folds test data.
    • Positive values model explains some variance.
    • Negative values model predictions are worse than simply predicting the mean of the test set.
  • RMSE is the average prediction error in hours of total sleep; lower values indicate more accurate predictions.

In our results: - Fold 2 (R 0.72, RMSE 2.62) The model predicted this heldout order well. - Fold 3 (R 1.84, RMSE 2.81) The model predicted very poorly; worse than the mean.

The cv_summary table averages performance across folds: - Mean_R2 0.03 On average, the model explains almost none of the variance in unseen orders.
- SD_R2 1.06 R varies widely across folds the model works well for some orders, poorly for others.
- Mean_RMSE 3.03 Predictions are off by about 3 hours of total sleep on average.
- SD_RMSE 0.50 RMSE is relatively consistent across folds.

Key takeaway:
The models ability to generalize to entirely new orders is weak. Some groups are predicted well, but others very poorly likely because important predictors of differences between orders are missing from the fixed effects.
